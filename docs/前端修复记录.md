# 前端修复记录：移除 Plots 引用

## 问题描述

前端组件仍然引用已删除的 `plots` 系统，导致运行时错误：
- `world.plots` 字段不存在
- 事件类型显示过时（plot_created, plot_triggered 等）
- Agent 显示引用不存在的 `role` 字段

### 4. knowledge-graph.ts

**修改前**:
```typescript
// 3. 添加剧情作为节点
for (const plot of world.plots) {
  graph.addNode({
    id: plot.id,
    type: 'plot',
    label: plot.title,
    properties: { ... },
  })
  
  // 连接剧情与主角/对手/配角
  for (const protagonistSeed of plot.protagonists) { ... }
  for (const antagonistSeed of plot.antagonists) { ... }
  for (const supportingSeed of plot.supporting) { ... }
}
```

**修改后**:
```typescript
// 3. 添加叙事作为节点（涌现式叙事系统）
for (const narrative of world.narratives.patterns) {
  graph.addNode({
    id: narrative.id,
    type: 'concept',
    label: narrative.type,
    properties: {
      type: narrative.type,
      intensity: narrative.intensity,
      participants: narrative.participants,
      context: narrative.context,
      status: narrative.status,
    },
  })

  // 连接叙事与参与者
  for (const participantSeed of narrative.participants) {
    graph.addEdge({
      id: `${narrative.id}-participant-${participantSeed}`,
      source: participantSeed,
      target: narrative.id,
      relation: 'participates_in',
      weight: narrative.intensity,
    })
  }
}
```

**改进点**:
- 使用叙事模式（narrative patterns）替代剧情
- 参与者关系而非固定角色
- 叙事强度作为边的权重

## 系统架构对比

### 旧架构（自上而下）
```
司命系统 → 生成剧情 → 分配角色（主角/配角/NPC）→ Agents 执行剧情
```

### 新架构（涌现式）
```
Agents 自由互动 → 叙事识别系统检测模式 → 故事弧自然形成 → 多视角总结
```
