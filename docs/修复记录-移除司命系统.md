# 修复记录：移除司命系统引用

## 问题描述

在 Phase 4 实现涌现式叙事系统时，我们移除了自上而下的司命（Siming）系统，但有些文件仍然引用了已删除的司命模块，导致编译错误：

```
Module not found: Can't resolve '@/server/llm/siming-nuwa-coordinator'
```

## 修复内容

### 1. 删除过时的 API 路由

删除了两个引用司命系统的 API 路由：

- `app/api/plots/coordinate/route.ts` - 引用 `siming-nuwa-coordinator`
- `app/api/plots/generate/route.ts` - 引用 `siming-generator`

这些 API 在涌现式叙事系统中不再需要，因为：
- 不再有自上而下的剧情生成
- 故事从 agents 的自由互动中自然涌现
- 叙事识别系统会自动检测涌现的叙事模式

### 2. 修复世界初始化路由

**文件**: `app/api/worlds/initialize/route.ts`

**修改前**:
```typescript
import { createNuwaService } from '@/engine/nuwa-service'

// 使用简单的 nuwa-service 创建 agents
const nuwa = createNuwaService({ ... })
for (let i = 0; i < agentCount; i++) {
  const result = nuwa.createAgent({ ... })
  if (result.agent) {
    newAgents.push(result.agent)
  }
}
```

**修改后**:
```typescript
import { generatePersonalAgents } from '@/server/llm/agent-generator'

// 使用 LLM 生成个性化 agents
const newAgents = await generatePersonalAgents({
  prompt: `${world.environment.description}\n\n社会背景：${JSON.stringify(world.social_context)}`,
  count: agentCount,
})
```

**改进点**:
- 使用 LLM 生成更丰富、更个性化的 agents
- Agents 具有动态生成的职业、说话风格、做事方式、专长、核心信念
- 更符合涌现式设计理念

### 3. 修复知识图谱类型引用

**文件**: `src/engine/knowledge-graph.ts`

**修改**:
```typescript
// 移除
import type { PlotArc } from '@/domain/siming'

// PlotArc 类型已不存在，因为司命系统已被移除
```

## 系统架构变化

### 旧架构（自上而下）
```
盘古创世 → 女娲造人 → 司命编织剧情 → Agents 执行剧情
```

### 新架构（涌现式）
```
盘古创世 → 女娲造人（LLM 生成个性化 agents）→ Agents 自由互动 → 叙事自然涌现
                                                    ↓
                                            叙事识别系统检测模式
                                                    ↓
                                            故事弧检测器分析结构
                                                    ↓
                                            叙事总结器生成摘要
```
