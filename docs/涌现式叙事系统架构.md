# 涌现式叙事系统架构设计

## 核心问题分析

### 当前系统的根本矛盾

1. **司命系统的自上而下控制 vs 真正的自主演化**
   - 司命预设剧情 = 强制叙事走向
   - 真正的涌现 = 从 agent 交互中自然产生故事
   - 这是根本性的架构冲突

2. **知识图谱的局限性**
   - 当前：静态关系图，结构化存储
   - 问题：缺乏语义理解、推理能力、动态演化
   - 需要：GraphRAG + 语义层 + 推理引擎

3. **推荐系统的表面化**
   - 当前：基于关键词匹配的简单推荐
   - 问题：无法捕捉深层语义和隐含关系
   - 需要：基于 embedding 的语义推荐 + 协同过滤

## 新架构设计：涌现式叙事引擎

### 1. 移除司命系统，引入涌现式叙事

#### 司命系统的问题
```
❌ 预设剧情弧 → 强制 agents 按剧本行动
❌ 触发条件 → 人为控制叙事节点
❌ 成功/失败条件 → 限制可能性空间
❌ 主角/对手/配角 → 固化角色关系
```

#### 涌现式叙事的替代方案
```
✅ 事件流 → agents 自由交互产生事件
✅ 叙事识别 → 从事件中识别叙事模式
✅ 故事弧涌现 → 自然形成起承转合
✅ 角色关系动态 → 从互动中自然演化
```

### 2. 新架构：三层系统

```
┌─────────────────────────────────────────────────────────┐
│                    叙事层 (Narrative Layer)              │
│  - 叙事识别器 (Narrative Recognizer)                     │
│  - 故事弧检测器 (Story Arc Detector)                     │
│  - 叙事总结器 (Narrative Summarizer)                     │
└─────────────────────────────────────────────────────────┘
                            ↑
                    事件流 (Event Stream)
                            ↑
┌─────────────────────────────────────────────────────────┐
│                  仿真层 (Simulation Layer)               │
│  - Agent 自主决策 (Autonomous Decision Making)           │
│  - 环境交互 (Environment Interaction)                    │
│  - 社交网络 (Social Network)                             │
│  - 推荐系统 (Recommendation System)                      │
└─────────────────────────────────────────────────────────┘
                            ↑
                    状态查询 (State Query)
                            ↑
┌─────────────────────────────────────────────────────────┐
│                  知识层 (Knowledge Layer)                │
│  - GraphRAG (语义图谱)                                   │
│  - 向量数据库 (Vector DB)                                │
│  - 记忆系统 (Memory System)                              │
│  - 推理引擎 (Reasoning Engine)                           │
└─────────────────────────────────────────────────────────┘
```

### 3. 核心组件重新设计

#### 3.1 GraphRAG 替代简单知识图谱

**当前知识图谱的问题**:
- 只存储结构化关系
- 无法理解语义
- 无法推理隐含关系
- 静态更新

**GraphRAG 设计**:
```typescript
type GraphRAGNode = {
  id: string
  type: NodeType
  label: string
  properties: Record<string, any>
  
  // 语义层
  embedding: number[]  // 768 维向量
  semantic_tags: string[]  // 语义标签
  
  // 时序层
  created_at: number
  updated_at: number
  version: number
  history: NodeSnapshot[]
  
  // 推理层
  inferred_properties: Record<string, any>  // 推理出的属性
  confidence_scores: Record<string, number>  // 置信度
}

type GraphRAGEdge = {
  id: string
  source: string
  target: string
  relation: RelationType
  
  // 语义层
  semantic_strength: number  // 语义相似度
  embedding: number[]
  
  // 动态层
  weight: number  // 动态权重
  temporal_pattern: number[]  // 时间模式
  
  // 推理层
  inferred: boolean  // 是否为推理出的关系
  evidence: string[]  // 证据链
  confidence: number
}
```

**GraphRAG 能力**:
1. **语义搜索**: 通过 embedding 找到语义相关的节点
2. **关系推理**: 从已知关系推理出隐含关系
3. **时序演化**: 追踪关系的动态变化
4. **多跳查询**: 复杂的图查询和路径发现
5. **社区发现**: 自动识别 agent 群体和派系

#### 3.2 涌现式叙事识别器

**不再预设剧情，而是识别涌现的叙事模式**:

```typescript
type NarrativePattern = {
  type: 'conflict' | 'romance' | 'alliance' | 'betrayal' | 'discovery' | 'transformation'
  participants: string[]  // agent IDs
  events: WorldEvent[]
  intensity: number  // 叙事强度
  emotional_arc: number[]  // 情感曲线
  started_at: number
  status: 'emerging' | 'developing' | 'climax' | 'resolving' | 'concluded'
}

class NarrativeRecognizer {
  /**
   * 从事件流中识别叙事模式
   */
  async recognizePatterns(events: WorldEvent[], graph: GraphRAG): Promise<NarrativePattern[]> {
    // 1. 事件聚类：找到相关的事件序列
    const eventClusters = await this.clusterEvents(events)
    
    // 2. 模式匹配：识别叙事原型
    const patterns: NarrativePattern[] = []
    
    for (const cluster of eventClusters) {
      // 冲突模式：两个 agents 之间的对抗
      const conflicts = this.detectConflicts(cluster, graph)
      patterns.push(...conflicts)
      
      // 联盟模式：agents 形成合作关系
      const alliances = this.detectAlliances(cluster, graph)
      patterns.push(...alliances)
      
      // 转变模式：agent 的重大改变
      const transformations = this.detectTransformations(cluster, graph)
      patterns.push(...transformations)
      
      // 发现模式：揭示重要信息
      const discoveries = this.detectDiscoveries(cluster, graph)
      patterns.push(...discoveries)
    }
    
    return patterns
  }
  
  /**
   * 检测冲突模式
   */
  private detectConflicts(events: WorldEvent[], graph: GraphRAG): NarrativePattern[] {
    const conflicts: NarrativePattern[] = []
    
    // 查找对抗性互动
    for (let i = 0; i < events.length - 1; i++) {
      const e1 = events[i]
      const e2 = events[i + 1]
      
      // 检查是否涉及相同的 agents
      const agents1 = this.extractAgents(e1)
      const agents2 = this.extractAgents(e2)
      const commonAgents = agents1.filter(a => agents2.includes(a))
      
      if (commonAgents.length >= 2) {
        // 检查情感极性是否相反
        const sentiment1 = this.analyzeSentiment(e1)
        const sentiment2 = this.analyzeSentiment(e2)
        
        if (sentiment1 * sentiment2 < 0) {  // 相反极性
          conflicts.push({
            type: 'conflict',
            participants: commonAgents,
            events: [e1, e2],
            intensity: Math.abs(sentiment1 - sentiment2),
            emotional_arc: [sentiment1, sentiment2],
            started_at: e1.timestamp,
            status: 'emerging'
          })
        }
      }
    }
    
    return conflicts
  }
  
  /**
   * 追踪叙事发展
   */
  async trackNarrativeDevelopment(
    pattern: NarrativePattern,
    newEvents: WorldEvent[]
  ): Promise<NarrativePattern> {
    // 检查新事件是否延续这个叙事
    const relevantEvents = newEvents.filter(e => 
      this.isRelevantToNarrative(e, pattern)
    )
    
    if (relevantEvents.length === 0) {
      return pattern  // 叙事暂停
    }
    
    // 更新叙事状态
    const updatedPattern = {
      ...pattern,
      events: [...pattern.events, ...relevantEvents],
      emotional_arc: [
        ...pattern.emotional_arc,
        ...relevantEvents.map(e => this.analyzeSentiment(e))
      ]
    }
    
    // 判断叙事阶段
    updatedPattern.status = this.determineNarrativeStage(updatedPattern)
    
    return updatedPattern
  }
  
  /**
   * 判断叙事阶段
   */
  private determineNarrativeStage(pattern: NarrativePattern): NarrativePattern['status'] {
    const { emotional_arc, intensity } = pattern
    
    // 分析情感曲线的变化趋势
    const recentArc = emotional_arc.slice(-5)
    const variance = this.calculateVariance(recentArc)
    
    if (variance > 0.5 && intensity > 0.7) {
      return 'climax'  // 高强度 + 高波动 = 高潮
    } else if (variance < 0.2 && intensity < 0.3) {
      return 'resolving'  // 低强度 + 低波动 = 解决中
    } else if (emotional_arc.length < 3) {
      return 'emerging'  // 刚开始
    } else {
      return 'developing'  // 发展中
    }
  }
}
```

#### 3.3 语义推荐系统

**当前推荐系统的问题**:
- 基于关键词匹配，太表面
- 无法理解深层语义
- 无法发现隐含的兴趣

**语义推荐系统**:
```typescript
class SemanticRecommendationSystem {
  constructor(
    private vectorDB: VectorDatabase,
    private graphRAG: GraphRAG
  ) {}
  
  /**
   * 为 agent 推荐事件（基于语义）
   */
  async recommendEvents(
    agent: PersonalAgentState,
    world: WorldSlice
  ): Promise<WorldEvent[]> {
    // 1. 构建 agent 的兴趣向量
    const interestEmbedding = await this.buildInterestEmbedding(agent)
    
    // 2. 语义搜索相关事件
    const semanticMatches = await this.vectorDB.search(
      interestEmbedding,
      { topK: 50, threshold: 0.7 }
    )
    
    // 3. 基于社交网络的协同过滤
    const socialRecommendations = await this.collaborativeFiltering(agent, world)
    
    // 4. 基于叙事相关性的推荐
    const narrativeRecommendations = await this.narrativeBasedRecommendation(agent, world)
    
    // 5. 混合推荐
    return this.mergeRecommendations([
      { events: semanticMatches, weight: 0.4 },
      { events: socialRecommendations, weight: 0.3 },
      { events: narrativeRecommendations, weight: 0.3 }
    ])
  }
  
  /**
   * 构建 agent 的兴趣向量
   */
  private async buildInterestEmbedding(agent: PersonalAgentState): Promise<number[]> {
    // 收集 agent 的所有相关文本
    const texts = [
      agent.occupation || '',
      agent.core_belief || '',
      ...agent.goals,
      ...agent.expertise || [],
      ...agent.memory_long.map(m => m.content)
    ].filter(t => t.length > 0)
    
    // 生成 embedding 并平均
    const embeddings = await Promise.all(
      texts.map(t => this.vectorDB.embed(t))
    )
    
    return this.averageEmbeddings(embeddings)
  }
  
  /**
   * 协同过滤推荐
   */
  private async collaborativeFiltering(
    agent: PersonalAgentState,
    world: WorldSlice
  ): Promise<WorldEvent[]> {
    // 找到相似的 agents
    const similarAgents = await this.findSimilarAgents(agent, world)
    
    // 收集他们感兴趣的事件
    const interestedEvents = new Map<string, number>()
    
    for (const similarAgent of similarAgents) {
      const events = await this.getAgentInterestedEvents(similarAgent, world)
      for (const event of events) {
        const score = interestedEvents.get(event.id) || 0
        interestedEvents.set(event.id, score + similarAgent.similarity)
      }
    }
    
    // 排序并返回
    return Array.from(interestedEvents.entries())
      .sort((a, b) => b[1] - a[1])
      .slice(0, 20)
      .map(([id, _]) => world.events.find(e => e.id === id)!)
      .filter(e => e !== undefined)
  }
  
  /**
   * 基于叙事的推荐
   */
  private async narrativeBasedRecommendation(
    agent: PersonalAgentState,
    world: WorldSlice
  ): Promise<WorldEvent[]> {
    // 找到 agent 参与的叙事
    const agentNarratives = await this.graphRAG.findAgentNarratives(agent.genetics.seed)
    
    // 推荐这些叙事的最新事件
    const narrativeEvents: WorldEvent[] = []
    
    for (const narrative of agentNarratives) {
      const latestEvents = world.events.filter(e =>
        narrative.events.some(ne => ne.id === e.id)
      ).slice(-5)
      
      narrativeEvents.push(...latestEvents)
    }
    
    return narrativeEvents
  }
}
```

#### 3.4 记忆系统重新设计

**分层记忆架构**:
```typescript
type MemorySystem = {
  // 个体记忆
  personal: {
    episodic: EpisodicMemory[]  // 情节记忆（具体事件）
    semantic: SemanticMemory[]  // 语义记忆（知识和概念）
    procedural: ProceduralMemory[]  // 程序记忆（技能和习惯）
  }
  
  // 群体记忆
  collective: {
    shared_narratives: NarrativePattern[]  // 共享叙事
    cultural_knowledge: CulturalKnowledge[]  // 文化知识
    social_norms: SocialNorm[]  // 社会规范
  }
  
  // 记忆索引
  index: {
    temporal: TemporalIndex  // 时间索引
    semantic: SemanticIndex  // 语义索引
    social: SocialIndex  // 社交索引
  }
}

class MemoryManager {
  /**
   * 存储记忆时自动建立多维索引
   */
  async storeMemory(agent: PersonalAgentState, memory: MemoryRecord): Promise<void> {
    // 1. 生成 embedding
    const embedding = await this.embed(memory.content)
    
    // 2. 存储到向量数据库
    await this.vectorDB.insert({
      id: memory.id,
      vector: embedding,
      metadata: {
        agent_id: agent.genetics.seed,
        timestamp: memory.timestamp,
        importance: memory.importance,
        emotional_weight: memory.emotional_weight
      }
    })
    
    // 3. 更新 GraphRAG
    await this.graphRAG.addMemoryNode(agent.genetics.seed, memory)
    
    // 4. 检查是否形成群体记忆
    await this.checkCollectiveMemoryFormation(memory)
  }
  
  /**
   * 检查群体记忆形成
   */
  private async checkCollectiveMemoryFormation(memory: MemoryRecord): Promise<void> {
    // 查找相似的记忆
    const similarMemories = await this.vectorDB.search(
      await this.embed(memory.content),
      { topK: 10, threshold: 0.8 }
    )
    
    // 如果多个 agents 有相似记忆，形成群体记忆
    if (similarMemories.length >= 3) {
      const sharedNarrative = await this.synthesizeSharedNarrative(similarMemories)
      await this.storeCollectiveMemory(sharedNarrative)
    }
  }
}
```

## 关键技术选型

- 易于部署和扩展

- 高质量语义表示
- API 调用简单
- 成本可控

### 图数据库
**推荐：Neo4j 或保持当前内存实现**
- Neo4j：成熟的图数据库，强大的查询能力
- 内存实现：更快，但规模受限

### LLM
**保持当前选择**
- Claude 3.5 Sonnet：高质量推理
- 成本可控

## 预期效果

3. **更真实**：叙事从真实互动中涌现

3. **记忆质量**：从简单存储到智能索引
4. **叙事质量**：从预设剧本到自然涌现

### 用户体验
1. **更有趣**：每次都有新的惊喜
2. **更真实**：agents 行为更自然
3. **更深度**：可以探索复杂的社会现象
4. **更灵活**：可以注入不同的种子信息

## 总结

**司命系统应该被移除**，因为它代表了"自上而下的控制"，与"自下而上的涌现"根本矛盾。

**新架构的核心理念**：
1. 让 agents 自由互动
2. 从互动中识别叙事模式
3. 用语义技术理解深层含义
4. 让故事自然涌现

这才是真正的群体智能和涌现式叙事。
